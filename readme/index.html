開発環境の構築

KBC実行委員会のWebページではページの可搬性を高めるために、全てのパスを絶対パスで記述します。そのため、Webページの実際の見た目を確認しながらローカル環境のみで開発を進めるためにはいくつかの準備が必要となります。そうした準備の仕方については以下のリンク先を参照して下さい。

KBC Bootstrapの使用

KBC実行委員会のWebページはjQueryおよびTwitter BootstrapをベースとしたJavaScript/CSSライブラリである「KBC Bootstrap」によって構成されています。

KBC Bootstrapでは適切なクラスの指定によって、コンテンツを統一されたデザインで簡単に仕上げることができます。指定できるクラスの一覧とプレビュー、使い方の詳細については以下のリンク先を参照して下さい。

ディレクトリ構造の管理

KBC実行委員会のWebページは将来的にファイルが増大したとしても、その検索・維持・管理に大きなコストの掛かることがないよう、単純かつ明確なディレクトリ構造を持っています。そのため、新たなHTMLファイルや画像ファイル、CSSファイル、JavaScriptファイルに加え、応募書類などのダウンロードファイルを追加する場合は必ず以下のリンク先を参照し、その構造に従った名前付け、ディレクトリ作成を行って下さい。

記法の統一

特定のページに限定されたCSSやJavaScriptはそのページ専用のCSSファイルやJavaScriptファイルを作成して読み込むことで適用するべきですが、どこまでのデザインをHTMLで行いどこからをCSSで行うか、JavaScriptの関数などをどのように記述するか、使うべきでない構文などはあるか、といった点は概して曖昧になり、バグの原因となります。以下のリンク先ではそうならないための指針として、KBCのWebページで推奨されているHTML、CSS、JavaScriptの記法について解説しています。
